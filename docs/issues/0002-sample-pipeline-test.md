# Issue 2: サンプルパイプラインの動作確認テストを追加

## 概要
- BasicEngine と MemoryStore を利用し、サンプルのジョブ入力をパイプラインに流し込む統合テストを作成する。
- ジョブがキュー投入から完了まで正しく遷移し、結果にテキストのレスポンスが含まれることを検証する。
- `go test ./...` でテストが実行されるようにし、テストの実行手順を README に追記する。

## 受け入れ条件
- サンプルパイプラインの実行を再現する Go のテストコードが `internal/engine` 配下に追加されている。
- テストではジョブの最終状態が `succeeded` となり、`result.items` にテキストデータが含まれることを確認している。
- README にテストの目的と実行方法 (`go test ./...`) が日本語で記載されている。
